**GUÍA DE ESCRITURA PARA CENTRO DE AYUDA**

**Banca Personas y Empresas**

**🎯 OBJETIVO**

Definir cómo redactar artículos del Centro de Ayuda para resolver dudas del usuario de forma clara, rápida y autogestionable, reduciendo la necesidad de contacto con soporte.

**🧠 PRINCIPIOS GENERALES**

- El objetivo principal es **resolver en el primer contacto**
- El contenido debe impulsar la **autogestión del usuario**
- El lenguaje debe ser **claro, directo y accionable**
- Se debe evitar complejidad innecesaria o tecnicismos

**✍️ TONO Y ESTILO**

- Instructivo y funcional
- Frases cortas
- Lenguaje simple y sin redundancia
- Cercano, pero profesional
- Orientado siempre a la acción

**🧱 ESTRUCTURA DE UN ARTÍCULO**

Todo artículo del Centro de Ayuda debe seguir esta estructura:

**1\. TÍTULO**

El título debe responder a:

“¿Qué quiere lograr el usuario con esta guía?”

**Reglas:**

- Usar verbos en infinitivo o preguntas
- Incluir palabras clave del tema
- Máximo 12 palabras
- Puede usar formato pregunta o descriptivo

**Ejemplos:**

- ¿Cómo crear una Cuenta de Ahorros?
- Cómo cobrar un giro en ventanilla
- Pagos disponibles en Banco del Barrio

**2\. INTRODUCCIÓN**

**Función:**

Explicar en una sola idea qué hará el usuario.

**Reglas:**

- 1 oración clara
- Sin tecnicismos
- Debe indicar el objetivo del proceso
- Si hay pasos, incluir “Sigue estos pasos”

**Ejemplo:**

Con esta opción puedes crear un ahorro automático para cumplir tus metas.

**3\. REQUISITOS (SI APLICA)**

**Función:**

Indicar lo necesario antes de iniciar el proceso.

**Reglas:**

- Frases cortas
- Si no aplica, escribir: “No se requieren requisitos previos”

**Ejemplo:**

- Tener una cuenta activa
- Documento de identidad vigente

**4\. PASOS A SEGUIR**

**Reglas:**

- Iniciar indicando el canal (App / Web / etc.)
- Usar numeración (1, 2, 3…)
- Cada paso = una acción
- Verbos en imperativo:
    - Ingresa
    - Selecciona
    - Revisa
- Máximo 2 oraciones por paso
- Mantener consistencia en nivel de detalle

**Ejemplo:**

1.  Ingresa a la App del banco
2.  Selecciona “Metas”
3.  Configura el monto que deseas ahorrar

**5\. INFORMACIÓN ADICIONAL (OPCIONAL)**

Usar cuando se necesite complementar información.

**Formato:**

**Importante:**

- El valor mínimo es $X
- El tiempo de acreditación es de 24 horas

**6\. CASOS ESPECIALES**

Derivar a otros canales cuando:

- Existen errores técnicos no resolubles en el artículo
- El usuario no recibe códigos de seguridad o contraseñas
- Se requiere soporte de un oficial o especialista
- Procesos sensibles como desbloqueos o actualización de datos
- Casos que requieren asistencia personalizada

**Ejemplo:**

Si necesitas ayuda adicional, contáctanos por ChatBG para asistencia.

**🖼️ IMÁGENES**

- Usar solo si aportan valor real al proceso
- Evitar imágenes innecesarias en cada paso
- Optimizar peso (JPEG o WebP, máximo recomendado 200 KB)
- Ocultar siempre datos sensibles
- Nombrar archivos de forma descriptiva

**🔗 LINKS Y REDIRECCIONES**

- No usar URLs visibles
- Usar texto enlazado descriptivo
- El enlace debe indicar claramente su destino

**Ejemplo correcto:**

- Ingresa a tu perfil desde **Mi Cuenta**

**Ejemplo incorrecto:**

- https://banco.com/perfil

**📊 TABLAS**

- Usar solo cuando mejoren la comprensión
- Información estructurada o comparativa
- Mantener legibilidad en mobile y desktop
- Evitar tablas muy extensas o con demasiado texto

**📏 ESPACIADO Y FORMATO**

- Un espacio entre párrafos
- Separar viñetas con espacio
- Evitar saltos dobles innecesarios
- Mantener lectura limpia y escaneable

**🔐 CONTENIDO SENSIBLE**

Siempre ocultar o difuminar:

- Cédulas
- Números de cuenta
- Tarjetas
- Emails
- Saldos
- Datos personales

**🧭 BUENAS PRÁCTICAS DE UX WRITING**

- Priorizar verbos de acción
- Mantener estructura consistente en todos los artículos
- Usar negritas solo para datos clave
- Evitar saturación de información
- Guiar al usuario paso a paso

**🚫 EVITAR**

- Lenguaje ambiguo
- Tecnicalismos innecesarios
- Párrafos largos
- Falta de pasos claros
- Información sin jerarquía
- Instrucciones incompletas

**🧠 PRINCIPIO FINAL**

El Centro de Ayuda no explica procesos: los hace fáciles de completar.
